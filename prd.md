# 产品需求文档（PRD）

## 一、产品一句话定位

> **"2026世界杯中文对阵图预测器"**——面向中国球迷的世界杯淘汰赛预测和社交分享工具，自动将48队赛程转为北京时间，3分钟完成冠军预测，一键分享到朋友圈和小红书。

## 二、目标用户画像

| 用户类型 | 描述 | 核心需求 | 典型行为 |
|---------|------|---------|---------|
| **社交型球迷**（主力用户） | 22-35岁，男性为主，关注世界杯但不一定是深度球迷。小红书/微信朋友圈活跃用户。 | 做出自己的冠军预测并分享炫耀、和好友比拼预测准确率 | "我做了一个预测，巴西夺冠！你也来试试？" |
| **硬核球迷**（价值用户） | 25-40岁，熟悉各队实力，会认真分析对阵形势 | 深度数据分析、精确预测、看历史战绩 | 反复调整预测、研究每组出线形势 |
| **氛围型用户**（增量用户） | 18-30岁，女性比例较高，世界杯期间跟风参与 | 简单好玩的互动、好看的结果图、方便分享 | 随机点球队名、截图发朋友圈 |

## 三、核心功能列表

### P0（MVP必须有，没它用户不会用）

| 编号 | 功能 | 描述（给AI编程看，零歧义） |
|------|------|--------------------------|
| P0-0 | **随机预测「啊哈时刻」** | 页面加载时自动生成一套随机预测结果并展示动画——用户3秒内看到"冠军是谁"。展示后底部出现"不满意？自己来预测"和"分享这个结果"两个按钮。这是"引擎级设计原则——即时出结果"的实现。 |
| P0-1 | **48队分组展示** | 展示12个小组×4支队伍的完整表格。每行显示：国旗emoji + 中文队名 + FIFA排名。数据来源：静态JSON文件。手机端用卡片式布局，每张卡片一个小组。 |
| P0-2 | **小组排名预测** | 用户可调整每组4队的排名顺序。手机端：点击队伍→高亮→再点击另一个队伍交换位置。PC端：支持拖拽。每组前2名自动晋级32强淘汰赛，第3名进入"最佳第3名"候选池。 |
| P0-3 | **最佳第3名自动选拔** | 用户完成所有小组排名后，系统自动计算12个小组第3名的积分（小组内胜3分/平1分/负0分），按积分→净胜球→进球数排序，自动选取前8名晋级32强。用户无需手动选择（降低操作门槛）。 |
| P0-4 | **淘汰赛对阵图** | 基于小组预测+最佳第3名结果，自动生成32强完整对阵表。采用"逐步式"展示——手机端每次只显示当前一轮的对阵（左右对比卡片），用户选择胜者后动画过渡到下一轮。PC端可展示完整树状图。 |
| P0-5 | **预测结果展示** | 展示用户的完整预测：冠军（大字+国旗+动画）、亚军、四强、各组晋级情况。以一张可滚动的可视化图表呈现。 |
| P0-6 | **一键分享** | 生成一张精美分享海报（Canvas渲染HTML→PNG）。海报内容：顶部"我的2026世界杯冠军预测"标题 + 美观的冠军团队展示 + 对阵图缩略图 + 底部品牌水印"来自 世界杯预测器" + 二维码 + "你也来预测你的冠军→"。支持：保存图片、复制链接。**微信浏览器内使用WeChat JS-SDK的分享接口，普通浏览器使用Web Share API。** |
| P0-7 | **预测自动保存** | 所有预测进度自动保存到localStorage。用户关闭浏览器后再次打开，弹窗提示"你有未完成的预测，要继续吗？"（如有半成品）。 |

### P1（有了更好，提升留存和传播）

| 编号 | 功能 | 描述 |
|------|------|------|
| P1-1 | **示例预测展示** | 首页展示2-3个知名预测结果（如：知名KOL的预测、AI模拟预测），用户可点击查看详情。 |
| P1-2 | **好友竞猜联赛** | 用户创建"联赛"，设置昵称+密码（无需注册），生成邀请链接分享给好友。参与者无需注册，填昵称即可加入。所有人提交预测后根据实际比赛结果自动排名。（技术方案：数据存后端Supabase/Cloudflare D1） |

## 四、用户核心流程

### 主流程：首次用户 → 看到随机预测 → 自己做 → 分享

```
1. 用户打开页面（无需注册/登录）
2. 页面加载 → 自动生成随机预测 → 冠军动画展示 → 「3秒啊哈时刻」
3. 用户看到后有两个选择：
   A. 「分享这个结果」→ 直接跳到步骤7
   B. 「不满意，自己来预测」→ 进入步骤4
4. 用户进入小组排名调整（手机端点击交换，PC端拖拽）
5. 系统自动计算最佳第3名 → 进入淘汰赛对阵图
6. 逐轮选择胜者（逐步式展示，每次只显示当前一轮）
7. 完成 → 看到完整的预测结果页
8. 点击「分享我的预测」
9. 生成分享海报 → 保存图片 / 复制链接 / 分享到微信/小红书
```


### 回流流程：好友点击分享链接

```
1. 好友在朋友圈/小红书看到分享图片
2. 扫描二维码或点击链接 → 打开页面
3. 看到分享者的预测结果
4. 点击"我也来预测" → 进入主流程
5. 做完预测后（可选）加入分享者的联赛
```

## 五、成功指标定义

| 指标 | 目标值（MVP上线1周） | 测量方式 |
|------|-------------------|---------|
| 日活跃用户（DAU） | 1000+ | 埋点page_view |

| 好友联赛创建数 | 50+ 联赛 | 数据库统计 |

## 六、非功能需求

| 类别 | 要求 |
|------|------|
| **性能** | 首页加载<2秒，对阵图渲染<500ms |
| **移动端** | 移动端优先设计，完美适配微信内置浏览器 |
| **多平台** | 支持：微信浏览器、手机Safari/Chrome、桌面Chrome |
| **国际化** | 界面中文，数据支持多语言（球队名为中文） |
| **无障碍** | 至少支持屏幕阅读器的基础无障碍 |
| **SEO** | 页面需有完整的title/description/hreflang标签 |
| **安全** | 无用户登录/注册，无需处理敏感数据（MVP阶段） |

## 七、边界场景处理

| 场景 | 处理方式 |
|------|---------|
| 用户只完成小组预测未完成淘汰赛 | 自动保存到localStorage，下次打开恢复进度。未完成预测不生成分享图。 |
| 用户关闭浏览器后重新打开 | 检测localStorage中是否有未完成预测，弹窗「你有未完成的预测，要继续吗？」 |
| 最佳第3名出现积分相同 | 按FIFA规则：积分→净胜球→进球数→公平竞赛积分→抽签。简化版：积分→净胜球→进球数，仍相同则随机选取。 |
| 微信浏览器中分享 | 使用WeChat JS-SDK的分享接口（需配置公众号JS安全域名） |
| 非微信浏览器中分享 | 使用Web Share API，不支持则降级为「复制链接」+「保存图片」 |
| 小组赛开始后修改预测 | 小组赛首场比赛开球后，预测锁定不可修改。展示锁定提示。 |
| 加载失败/数据异常 | 展示友好的错误页面（中文提示+重试按钮），不裸奔 |

## 八、引擎级设计原则符合性

| 原则 | 实现方式 |
|------|---------|
| **即时可用（零门槛）** | 无需注册/登录，打开即用 |
| **即时出结果（3秒规则）** | P0-0：页面加载自动展示随机预测结果 |
| **产出可分享（自传播）** | P0-6：精美海报生成+微信/浏览器双通道分享 |
| **免费版「差一点」** | MVP阶段全免费。P2预留：分享图带水印、分辨率限制、联赛数量限制 |
